# kb-lint

## いつ使うか
- 週に 1 度、または手動で「知識の整理整頓」を指示されたとき
- 大量に raw/ を流し込んだ直後の品質確認

## 何をするか
1. `brain/wiki/` のすべてのファイルを Read する
2. 以下の 5 観点で issues を列挙する
   a. **矛盾**: 同じ概念が異なるページで違う定義 / 数値を持っている
   b. **欠落**: 複数ファイルで言及される固有名詞 / 概念に専用ページがない
   c. **重複**: 似た内容のページが複数あり、統合候補
   d. **数値不整合**: 数値の根拠が古い / 出典 raw が削除済み
   e. **新規候補**: 既存ページから派生して 1 ページ立てる価値のあるトピック
3. `brain/wiki/issues.md` を上書き生成 (フォーマットは下記)
4. ユーザーに対して「優先度高 3 件」を提案

## 出力フォーマット (wiki/issues.md)
# Lint Issues ({YYYY-MM-DD})

## 矛盾
- [ ] {issue 説明} (関連: [[page-a]], [[page-b]])

## 欠落
- [ ] {concept-name}: {N} ファイルで言及、専用ページなし

## 重複
- [ ] [[page-x]] と [[page-y]] の統合検討

## 数値不整合
- [ ] [[page-z]] の数値が raw/old.md に依存

## 新規候補
- [ ] {new-topic}: {既存ページからの派生理由}
