# kb-compile

## いつ使うか
- raw/ に新規ファイルが複数溜まったとき
- 「最近の素材を体系化したい」と感じたとき

## 何をするか
1. `brain/raw/` のすべてのファイルを Read する
2. 繰り返し登場する概念 (複数ファイルで言及) を列挙する
3. 各概念について、`brain/wiki/concepts/{concept-name}.md` を生成する
   - frontmatter に `sources: [raw/xxx.md]` を必ず書く
   - 定義 (1 段落)、キーインサイト (3-5 点)、関連ページへの `[[wikilink]]` を含める
4. `brain/wiki/index.md` に新しい概念を 1 行サマリ付きで追記
5. 生成・更新したファイル一覧をユーザーに報告

## 出力フォーマット
- 報告: 生成 N 件、更新 M 件、スキップ K 件
- 各ファイルへの絶対パス
- 次回の優先タスク提案
