# kb-report

## いつ使うか
- 「brain内の知識を使って〜を整理して」と問われたとき
- 過去のバックテスト結果や実装理由を調査するとき
- 日次・週次などのレポートを作成するとき

## 何をするか
1. 問いを受け取る
2. `brain/wiki/index.md` や関連しそうな概念ページを列挙・Read する
3. `brain/reports/{YYYY-MM-DD}_{topic-slug}.md` を新規作成
   - frontmatter に `used_wiki_pages: [...]` を必ず書く
   - 結論を冒頭に置く (3 行リード)
   - 根拠を構造化して並べる
   - 末尾に出典 wiki ページへのリンクを列挙
4. ファイルパスをユーザーに返す

## 出力フォーマット
- reports/ のファイルパス
- リードの 3 行 (チャットにも貼り出す)
- 使用した wiki ページの一覧

## 制約
- `raw/` のファイルは直接引用しない (必ず `wiki/` 経由)
- 出典のない断定をしない
- 既存の `daily/` フォルダのレポート形式に寄せる場合もあるが、保存先は `reports/` とすること
